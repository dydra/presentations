NXP : the Data Hub : storage technology features

  • Interoperability based on standards support: SPARQL 1.1 Query, Update, Federated Query, Graph Store HTTP Protocol:
    • Jena
    • Sesame
    • Linked Data API
    • YASGUI
  • All major RDF serialization formats, and others:
    • JSON-LD
    • Turtle
    • RDF/XML
    • RDF/JSON
    • N-Triples/ N-Quads
    • TriX, Pajek
    • Graphviz
  • First-class revisions, query access via SPARQL endpoint, temporal federation, a SPARQL revision clause extension.
  • Materialized views and strong cache identifiers, efficient remote access to named result sets
  • SPARQL extensions—implicit reification, collation, temporal operators, provenance records

NXP : the Data Hub : is the approach

  • Integrate representations for heterogeneous data sources/destinations
    • XML, together with XSLT : on import
    • XML, together with XSLT : through to recipient
A Graph Store import over HTTP is
curl -X PUT http://dydra.com/schema-org-test/schema-org/service \ -H "Content-Type: application/rdf+xml"
SPARQL request with XSLT via HTTP
curl http://dydra.com/schema-org-test/schema-org/sparql \ -H "Accept: application/sparql-results+xml"
becomes
curl 'http://dydra.com/schema-org-test/schema-org/sparql?style-sheet=some-stylesheet' \ -H "Accept: application/sparql-results+xml"

curl http://dydra.com/schema-org-test/schema-org/sparql -H "Accept: application/ld+json"

access x mode combinations
tree structured ordered HDT
archival backup VCF, LDF

A decimated structure would be declared as
pgsql { tr_crop_plots_seed { storage pgsql pgsql-table public.tr_crop_plots_seed } tr_crop_plots_seed_start { storage pgsql pgsql-table public.tr_crop_plots_seed_start } tr_crop_plots_seed_end { storage pgsql pgsql-table public.tr_crop_plots_seed_end } }
with the Postgres views defined as
CREATE OR REPLACE VIEW public.tr_crop_plots_seed AS SELECT NULL::text AS g, CONCAT('http://example.org/plot/', crop_plot) AS s, 'http://example.org/seed'::text AS p, seed AS o FROM farm.tr_crop_plots; CREATE OR REPLACE VIEW public.tr_crop_plots_seed_start AS SELECT NULL::text AS g, CONCAT('http://example.org/plot/', crop_plot) AS s, 'http://example.org/start_date'::text AS p, start_date AS o FROM farm.tr_crop_plots; CREATE OR REPLACE VIEW public.tr_crop_plots_seed_end AS SELECT NULL::text AS g, CONCAT('http://example.org/plot/', crop_plot) AS s, 'http://example.org/end_date'::text AS p, end_date AS o FROM farm.tr_crop_plots;

:location ; :media-type 'application/sparql-query' ] [ a :Bind ; :dataset 'product-cms/hub' ] [ a :Project ; :dataset 'product-cms/hub' ] ) ] [ a :Conditional ; # if an export is necessary, :consequent [ a :Compose; :steps ( # address both datasets in coordinated transactions [ a :Begin; :dataset 'product-cms/hub' ; :dataset 'product-cms/customer' ] [ a :Update ; :mode 'continue'; # without auto-commit :steps ( # to transfer the root product descriptions via a # federated query through the specific dataset to the hub [ a :Decode ;
