NXP : the Data Hub : the technology

Dydra : a cloud-based RDF database and SPARQL processor, delivered as a service (DaaS)

  • In development since 2009, incorporated in 2011. Technology team based in Berlin. GmbH since 2013.
  • Provides free hosting for ontologies and small Linked Data datasets (“GitHub for RDF data”), each repository is a SPARQL endpoint.
  • Commercial arrangements available for managed turnkey hosting of larger datasets on dedicated hardware in the cloud or on-site, with SLAs.

NXP : the Data Hub : storage technology features

  • Interoperability based on standards support: SPARQL 1.1 Query, Update, Federated Query, Graph Store HTTP Protocol:
    • Jena
    • Sesame
    • ELDA
    • YASGUI
  • All major RDF serialization formats, and others:
    • JSON-LD
    • Turtle
    • RDF/XML
    • RDF/JSON
    • N-Triples/ N-Quads
    • TriX, Pajek
    • Graphviz
  • First-class revisions, query access via SPARQL endpoint, temporal federation, a SPARQL revision clause extension.
  • Materialized views and strong cache identifiers, efficient remote access to named result sets
  • SPARQL extensions—implicit reification, collation, temporal operators, provenance records

NXP : the Data Hub : is the approach

  • Integrate representations for heterogeneous data sources/destinations

    • XML, together with XSLT : through to recipient
A Graph Store import over HTTP is
curl -X PUT http://dydra.com/schema-org-test/schema-org/service \ -H "Content-Type: application/rdf+xml"
SPARLQL request with XSLT via HTTP
curl http://dydra.com/schema-org-test/schema-org/sparql \ -H "Accept: application/sparql-results+xml"
becomes
curl 'http://dydra.com/schema-org-test/schema-org/sparql?style-sheet=some-stylesheet' \ -H "Accept: application/sparql-results+xml"
